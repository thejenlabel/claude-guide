핵심 한 줄
클로드는 기억하지 못한다. 잘 쓰는 사람과 못 쓰는 사람의 차이는 질문 실력이 아니라 구조를 깔아두느냐다.

한 번 구조를 만들면 클로드·클로드 코드·옵시디언이 같은 자료를 공유하며 일하는 직원이 된다. 그 구조의 중심은 GitHub 한 곳이다.

1전체 도구 지도 — 무엇이 무엇과 연결되나

도구는 4개의 "입력 엔진" + 1개의 "본진(GitHub)" + 보기용 미러로 나뉜다. 모든 .md 파일이 GitHub로 모이기 때문에 어떤 도구로 작업해도 충돌이 안 난다.

2각 도구는 한마디로 뭐냐

💬 Claude (Cowork)

데스크톱 앱. 내 컴퓨터 폴더를 읽고 엑셀·PPT·문서를 실제 파일로 만든다. 말로 시키면 일하는 직원.

역할 ▸ 일상 작업·문서·전략·리서치

⌨️ Claude Code

터미널(명령창)에서 도는 도구. GitHub repo를 통째로 보고 여러 파일을 한 번에 고치고 커밋한다. 자동화·코딩에 강함.

역할 ▸ repo 대량 작업·자동화·개발

🗂️ Obsidian

로컬 마크다운(.md) 노트 앱. 생각·자료를 연결해 쌓는 "두 번째 뇌". 파일이 .md라 GitHub와 100% 호환.

역할 ▸ 지식 축적·노트·연결

🔁 Hermes (자동화 파이프라인)

외부 콘텐츠(유튜브 등)를 자동으로 받아 옵시디언 볼트에 정리해 넣는 수집 자동화. 손으로 복붙할 일을 없앤다.

역할 ▸ 자료 자동 수집·적재

🐙 GitHub

모든 .md의 단일 원본. 옵시디언 볼트 = repo, 클로드 코드가 작업하는 곳, Cowork가 커밋하는 곳. 전부 여기서 만난다.

역할 ▸ 단일 원본(SSOT)·버전 관리·충돌 해결

🧩 어떻게 한몸이 되나

옵시디언 볼트 폴더 = GitHub repo로 지정하면 끝. 그 안의 .md를 클로드·클로드 코드·Hermes가 모두 같이 본다.

한 폴더, 세 도구, 한 원본

5CLAUDE.md 시작 템플릿 (복붙용)

repo 최상단에 이 파일 하나만 두면 클로드의 톤·품질이 통째로 바뀐다.
# 프로젝트 헌법

## 답변 원칙
- 결론 먼저, 짧고 강하게
- 선택지를 줄여 1순위 / 2순위 / 버릴 것으로 구분
- 추상적 조언 금지 → 항상 "오늘 할 일 / 이번 주 할 일"로 쪼갬

## 저장 원칙 (SSOT)
- 모든 결정·문서는 이 GitHub repo의 .md 에 둔다
- 원본은 GitHub 하나, 노션·드라이브는 보기용 미러
- 작업이 끝나면 바로 커밋해서 최신 상태를 박아둔다

## 금지사항
- 장황한 설명 금지
- 이론·요약으로 끝내지 말고 항상 실행·비용·리스크로 끝낼 것

## 기본 답변 포맷
1. 결론  2. 지금 제일 중요한 것  3. 실행 순서 3단계
4. 버릴 것  5. 주의할 점

6매일 쓰는 루틴 — 이 3개만 습관화

1
시작할 때

"오늘 작업 시작" → 클로드가 context.md를 읽고 어제 어디까지 했는지 복기.

2
결정·인사이트가 나오면

"이거 memory에 기록하고 커밋해" → 흘려보내지 않고 DB에 쌓음.

3
자료가 생기면

링크를 던지고 "전략으로 바꿔줘" → 요약이 아니라 실행 포인트로 변환.

7흔한 실수 5가지

  • 매번 처음부터 설명 → CLAUDE.md·지식에 한 번 넣어두면 끝.
  • 결정을 채팅에 흘려보냄 → GitHub repo의 memory.md에 커밋. 안 그러면 사라진다.
  • 원본을 드라이브·노션·로컬에 동시에 둠 → 원본은 GitHub 하나, 나머지는 미러.
  • 모든 자료를 한 번에 다 넣기 → 필요한 것만(컨텍스트 다이어트). 많으면 품질 떨어짐.
  • 자료를 요약만 시키고 끝 → "그래서 뭘 해야 해?"까지 시켜야 돈이 된다.
